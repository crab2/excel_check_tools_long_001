# 行业规模 Excel 核对工具

一个原生 Rust 桌面应用。它使用可信的四级行业代码和可配置的企业划型标准，核对源工作簿 C 列的大中小微分类，并把修改或跳过原因写入可配置的标注列。

## 默认数据

应用编译时内置以下两份默认配置，首次启动无需手工选择：

- `docs/标准 更新.xlsx`：导入 23 条代码规则和 1 条“其他行业”规则。
- `docs/行业分类代码注释.xls`：导入 1,381 条四级行业细类及其一至三级归属。

用户在“配置中心”修改并保存后，应用优先读取本机配置，不会在下次启动时被内置文件覆盖。配置页也可以重新导入工作簿或恢复内置默认值。

## 核对规则

源工作簿应为 `.xlsx`，固定读取：

| 列 | 用途 |
|---|---|
| B | 账号名称，仅用于结果预览 |
| C | 当前大中小微分类，核对后按需修改 |
| E | 可信四级行业代码 |
| G | 员工人数，单位为人 |
| H | 资产总额，源数据单位为元 |
| I | 营业收入，源数据单位为元 |

应用忽略可能不准确的 D、F 列。它先通过 E 列在行业细类配置中找到四级至一级代码，再按“最具体代码优先”匹配标准表 B 列，例如：

```text
F5151 -> F515 -> F51 -> F
K7010 -> K701 -> K70 -> K
C3833 -> C383 -> C38 -> C
```

标准金额单位为万元，因此 H、I 在比较前除以 `10,000`。多指标规则从大型、中型、小型依次判断，只有该档全部必要指标达到下限才命中，否则继续下调，最终为微型。

以下情况不会改写 C 列：

- E 列代码为空或不在行业细类配置中；
- 没有匹配到划型标准；
- 规则要求的员工人数、资产总额或营业收入为空/非法。

默认标注列为 Q，可在数据核对页改为 J 之后的任意空列。应用始终导出到新的 `.xlsx` 文件，不覆盖源文件。

## 功能

- 数据核对：文件选择、后台核对、汇总统计、状态筛选、逐行预览和结果导出。
- 划型标准：搜索、新增、编辑、删除行业规则及其指标阈值。
- 行业细类：搜索、新增、编辑、删除四级代码及完整祖先链。
- 配置导入：从新的 `.xlsx` 标准表或 `.xls/.xlsx` 行业表重新导入。
- 本地持久化：规则、行业细类、标注列和最近使用路径分别保存。

## 开发运行

环境要求：Rust stable（MSVC 工具链）。

```powershell
cargo test
cargo run
```

发布构建：

```powershell
cargo build --release
```

可执行文件输出到 `target/release/industry-excel-checker.exe`。

## Windows MSI 发布

仓库包含 GitHub Actions 发布工作流。推送格式为 `vMAJOR.MINOR.PATCH` 的 tag 后，工作流会在 Windows runner 上：

1. 校验 tag 版本与 `Cargo.toml` 中的版本一致；
2. 构建 release 可执行文件；
3. 使用 WiX Toolset 生成 x64 MSI 安装包；
4. 验证 MSI 中包含可执行文件；
5. 上传 Actions Artifact，并创建带 MSI 和 SHA-256 校验文件的 GitHub Release。

例如发布 `0.1.0`：

```powershell
git tag v0.1.0
git push origin v0.1.0
```

如果要发布新版本，请先同步修改 `Cargo.toml` 中的 `package.version` 并提交，再创建对应 tag。安装包会命名为 `industry-excel-checker-<version>-x64.msi`。
